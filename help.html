Python Application Systemd Service Setup

This guide will help you create a systemd service for your Python application running in a virtual environment.

1. Create the Service File

Create a new service file at /etc/systemd/system/panel.service:

sudo nano /etc/systemd/system/panel.service

2. Service Configuration

Paste the following content, adjusting paths as needed:

[Unit]
Description=Python Panel Application
After=network.target

[Service]
User=root
Group=root
WorkingDirectory=/root/panel
Environment="PATH=/root/panel/venv/bin:/usr/local/sbin:/usr/local/bin:/usr/sbin:/usr/bin:/sbin:/bin"
ExecStart=/root/panel/venv/bin/python /root/panel/app.py
Restart=always
RestartSec=3

[Install]
WantedBy=multi-user.target
Important Notes:
  • Adjust /root/panel to your actual application directory
  • The Environment line ensures the virtual environment's Python is used
  • Restart=always will automatically restart your app if it crashes
  • The service will run as root (as your command suggests), but consider creating a dedicated user for security

3. Enable and Start the Service

# Reload systemd to recognize the new service sudo systemctl daemon-reload # Start the service sudo systemctl start panel # Enable it to start on boot sudo systemctl enable panel # Check status sudo systemctl status panel

4. Common Service Commands

# Stop the service
sudo systemctl stop panel

# Restart the service
sudo systemctl restart panel

# View logs
journalctl -u panel -f

# Disable automatic startup
sudo systemctl disable panel
Tip: If your application requires any environment variables, you can add them under the [Service] section with additional Environment= lines.
